Every `@flaky-tests/*` package is published to **both [npm](https://www.npmjs.com/org/flaky-tests)
and [JSR](https://jsr.io/@flaky-tests)**. Pick whichever registry fits your
workflow — the code is identical.

A working setup needs exactly **one runner plugin** (`plugin-bun` _or_ `plugin-vitest`) plus
**one store adapter** (`store-sqlite`, `store-turso`, `store-supabase`, or `store-postgres`).
The `core` package ships the CLI and is pulled in as a peer — install it explicitly if you
want to run `flaky-tests` directly.

<Aside type="tip" title="Not sure what to pick?">
See [Choosing a store](/flaky-tests/guides/choosing-a-store/) for the store comparison,
or jump to the [Quick Start](/flaky-tests/getting-started/quick-start/) for an end-to-end walkthrough.
</Aside>

## Core

The CLI + shared types. Installed automatically as a peer of every plugin, but you'll want
it explicitly if you run `flaky-tests --html` or `flaky-tests --prompt` from your own scripts.

<Tabs syncKey="registry">
  <TabItem label="npm">
    <PackageManagers pkg="@flaky-tests/core" dev />
  </TabItem>
  <TabItem label="JSR">
    ```sh
    deno add jsr:@flaky-tests/core
    npx jsr add @flaky-tests/core
    bunx jsr add @flaky-tests/core
    pnpm dlx jsr add @flaky-tests/core
    yarn dlx jsr add @flaky-tests/core
    ```
  </TabItem>
</Tabs>

## Runner plugins

Pick **one**. These wire into your test runner and capture failures into the store.

### `@flaky-tests/plugin-bun`

Bun test preload. Registers a global lifecycle hook and writes to the store via `bun:sqlite`.

<Tabs syncKey="registry">
  <TabItem label="npm">
    <PackageManagers pkg="@flaky-tests/plugin-bun" dev />
  </TabItem>
  <TabItem label="JSR">
    ```sh
    deno add jsr:@flaky-tests/plugin-bun
    bunx jsr add @flaky-tests/plugin-bun
    ```
  </TabItem>
</Tabs>

See [Setting up with Bun](/flaky-tests/guides/bun/) for the `bunfig.toml` preload step.

### `@flaky-tests/plugin-vitest`

Vitest reporter implementing `onInit` / `onFinished`. Works with Vitest 1.x+.

<Tabs syncKey="registry">
  <TabItem label="npm">
    <PackageManagers pkg="@flaky-tests/plugin-vitest" dev />
  </TabItem>
  <TabItem label="JSR">
    ```sh
    npx jsr add @flaky-tests/plugin-vitest
    pnpm dlx jsr add @flaky-tests/plugin-vitest
    yarn dlx jsr add @flaky-tests/plugin-vitest
    bunx jsr add @flaky-tests/plugin-vitest
    ```
  </TabItem>
</Tabs>

See [Setting up with Vitest](/flaky-tests/guides/vitest/) for the `vitest.config.ts` reporter registration.

## Store adapters

Pick **one**. All adapters implement the same [`IStore` interface](/flaky-tests/reference/istore/),
so swapping later only requires an environment variable change.

### `@flaky-tests/store-sqlite` — local

Local libSQL / SQLite, runs on Node and Bun. Zero-config, perfect for local dev.

<Tabs syncKey="registry">
  <TabItem label="npm">
